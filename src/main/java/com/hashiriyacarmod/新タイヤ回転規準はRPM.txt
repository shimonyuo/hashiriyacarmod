核心となる概念：スリップ率
現実の車では、タイヤの回転速度と車体の移動速度は必ずしも一致しないです。
スリップ率 = (タイヤの周速度 - 車体速度) / 車体速度

スリップ率 0% → グリップ走行（タイヤと地面が完全に同期）
スリップ率 100%以上 → 完全空転（タイヤだけ回って車は動かない）
スリップ率 -100% → ロック（ブレーキでタイヤ停止、車は滑る）


タイヤ回転速度の基準は「エンジン回転数」です
タイヤ回転速度
= エンジン回転数 ÷ ギア比 ÷ デフ比
エンジン回転数を一次情報として持ち、そこからタイヤ回転速度を計算します。車体速度は二次的な結果です。

ドリフト実装の骨格：
毎ティックの計算順序

① エンジン回転数を計算
   アクセル開度 → エンジントルク → RPM変化

② タイヤの周速度を計算
   RPM × タイヤ半径 × 2π ÷ ギア比

③ スリップ率を計算
   (タイヤ周速度 - 車体速度) / max(車体速度, 最小値)

④ グリップ力を計算
   スリップ率が小さい → グリップ大（地面をつかむ）
   スリップ率が大きい → グリップ小（滑る）
   ※ 現実ではμ-スリップ曲線（パチェカ曲線）で表現

⑤ 駆動力を計算
   グリップ力 × タイヤが地面を蹴る力

⑥ 車体速度を更新
   駆動力 → 加速度 → 速度変化

⑦ タイヤ描画の回転角度を更新
   タイヤ周速度をそのまま使う（スリップしてれば車速より速く回る）

描画への反映：
java// wheelSpinAngle は毎ティック加算していく
wheelSpinAngle += tireCircumferentialSpeed * 20f; // 20f は1秒20tick換算
// これをposeStackに渡す
poseStack.mulPose(Axis.XP.rotationDegrees(wheelSpinAngle));
tireCircumferentialSpeed（タイヤ周速度）を基準にするのがポイントです。車体速度ではなく、エンジンからギアを通じてタイヤに伝わった回転速度を使います。

スリップ率とグリップの関係（簡易版）：
javafloat slipRatio = (tireSpeed - vehicleSpeed) / Math.max(vehicleSpeed, 0.01f);

float grip;
if (Math.abs(slipRatio) < 0.1f) {
    // スリップ率10%以下 → ほぼフルグリップ
    grip = 1.0f;
} else if (Math.abs(slipRatio) < 0.3f) {
    // 10〜30% → グリップ低下（ドリフト域）
    grip = 1.0f - (Math.abs(slipRatio) - 0.1f) * 3.0f;
} else {
    // 30%以上 → ほぼ空転
    grip = 0.2f;
}

Immersive Vehiclesが「速度基準」だった問題の答え：
車体速度だけを見ていたので、タイヤと車体の「ズレ」という概念がなかったんです。エンジン回転数を一次情報として持ち、そこからタイヤ速度と車体速度の差分でスリップ率を計算する構造にすれば、空転・ドリフト・スタートダッシュ全部自然に表現できます。
