【フォルダ構造】
マイクラフォルダ\hashiriyacarpack（固定）\パックごとの名前フォルダ\クリエイティブタブごとの名前フォルダ\jsons・objs・textures（固定）\車やパーツごとのデータ.json・.obj・.png もしくはサブフォルダ
・hashiriyacarpack（固定）にパックフォルダを入れる場合、zipファイルだと認識されません。解凍した状態で入れてください。

・パック読み込み時、.jsonと拡張子以外の名前が同じ.obj、.pngを自動で紐づけるため、絶対に車両・パーツごとで名前を一致させてください。
（例：test1.json、test1.obj、test1.png）

・jsons・objs・textures（固定）下に自由な名前のサブフォルダを配置できます。json、obj、pngすべて同じサブフォルダ名で配置しないと、読み込みで自動紐づけできません。

・クリエイティブインベントリに追加されるアイテムタブには、直下→サブフォルダ（フォルダごとに区切る）でアイテムが名前順で並びます。
（例えば　直下に「test2」sub1フォルダに「test1・test3」sub2フォルダに「test0・test4」　の構造だった場合
アイテムタブでは、上から順に　「test2・test1・test3・test0・test4」　この順でアイテムが追加されるはずです、順番整理にうまいこと使ってください。）

・エンティティやアイテムのidにパックフォルダとタブフォルダの名前が自動で入るため、別パックとの重複対策になっています。しかし、安全のため基本的にパック内で同じ名前の車両やパーツを作らないでください。
名前を設定できるフォルダやファイルは独自性のある名前を付けることをおすすめします。

【車本体の.jsonテキスト】

"basic": {
    "type": "cars"　（車本体と定義。アイテムは地面を右クリックすることで車本体エンティティをスポーンさせられます。）
    "name": "Test Car3",　（車アイテムの名前を設定。）
（obj描写の描写距離。未記載の場合128に自動設定）
    "width": 2.0,　（コリジョンボックスの横サイズを設定。）
    "height": 1.5,　（コリジョンボックスの高さを設定。）
	
  }

【基本追加アイテム】
・レンチ
レンチを持ちエンティティのOBBボックスを向き右クリックで、パーツメニューを開けます。
レンチを持ちしゃがみ左クリックで、車両をアイテム化しインベントリに収納できます。アイテムドロップではありません。

【.objでのxyz座標をマインクラフト世界でのxyz座標に変換】

x→x y→-z z→y
blender上の「y座標はマイナス掛けてzの座標に」「z座標はそのままy座標に」書き込む

"hitbox": {
    "vertices": [
      [-1.01, 0.7, -2.1], // 0: 下面 初回x-、y+側の頂点
      [ 1.01, 0.7, -2.1], // 1: 下面 前回からx+側の頂点
      [ 1.01, 0.7,  1.4], // 2: 下面 前回からy-側の頂点
      [-1.01, 0.7,  1.4], // 3: 下面 前回からx-側の頂点 下面終了
      [-0.67, 1.2, -0.69], // 4: 上面 初回x-、y+側の頂点
      [ 0.67, 1.2, -0.69], // 5: 上面 前回からx+側の頂点
      [ 0.67, 1.2,  0.5], // 6: 上面 前回からy-側の頂点
      [-0.67, 1.2,  0.5]  // 7: 上面 前回からx-側の頂点 下面終了
    ] //blender内表記での案内です。blenderの頂点数値を「x座標はそのままxに」「y座標はマイナス掛けてzの座標に」「z座標はそのままy座標に」
  }
